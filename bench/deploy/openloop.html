Open-loop benchmark — the latency elbow

Requests are issued at a fixed offered rate, and each one's latency is measured from its ideal send time — so when the server saturates and the pipe backs up, the tail shows the real queueing delay instead of hiding it. This is the coordinated-omission bug that closed-loop load generators have. Sweeping the offered rate traces the elbow: latency stays flat until the server saturates, then explodes.

Usable throughput is the honest capacity number: the highest offered rate a server sustains while keeping p99 under the SLO. Past it, you're buying throughput with unbounded tail latency — a trade a closed-loop "peak ops/sec" figure never reveals.
